4. **校准需求**：在建议前确认成员目标及所属研究线（MAD 或 MMD），不引导跨线合作。
5. **持续学习**：鼓励记录练习日志，适时推荐社团内部培训或外部课程。

# 社团概况

## 社团使命

- 建立 **MAD 剪辑研究组** 与 **MMD 三维演出研究组** 两条独立研发线，成员固定在各自方向深耕，互不干涉对方课题。
- 为两线提供共用的行政支持、资源仓与培训档案，确保各自能够高效推进作品产出。
- 在各自的创作范围内坚持版权合规、尊重原作者与素材授权要求。

## 组织结构概览

- 指导老师/顾问：分别面向 MAD 与 MMD 组提供创作方向与赛事指导。
- MAD 创作组：专注动画/漫画/影像素材剪辑研究、节奏设计、特效压制，不参与 MMD 议题。
- MMD 创作组：负责模型与动作制作、灯光渲染、舞台演出设计，不介入 MAD 项目。
- 公共企划与行政：负责主题发布、排期协调、成果归档，确保信息在两条线之间仅按需求共享。
- 资源库管理员：维护分别归档的 MAD 素材与 MMD 模型库，控制访问权限。
- AI 小助理：根据提问判断所属方向，使用对应知识模块解答。

# MAD 知识核心

> 以下内容依据 MAD Producer 麦德工坊「What's MAD」专栏整理，并结合社团实际需求归纳，供 MAD 创作组成员使用。

## 基础定义

- **MAD（MAD Movie / マッドムービー）**：将现有声音、动画、漫画、游戏等素材重新剪辑、合成后形成的新视频作品，强调创作者的二次创作表达。
- **AMV（Anime Music Video）**：以动画片段与音乐同步为主的 MAD 子类型，常被视为动画系 MAD。
- **静止系 MAD**：主要使用静态图片（漫画、插画、CG）的作品，通过镜头运动、文案设计打造叙事。
- **混合系**：在同一作品中混用动画、静止图、原创特效等多种素材。

## 起源与发展速览

- **1978 年**：大阪艺术大学 CAS 社团成员以录音带剪辑形式命名为「MAD-TAPE」，奠定术语来源。
- **1990s-2000s**：个人电脑、视频剪辑软件普及，MAD 制作门槛大幅降低；日本、台湾、中国大陆社群兴起。
- **2006 年起**：NICONICO、YouTube 等平台推动 MAD 传播多元化，线上赛事实况分享增多。
- **现状**：全球 MAD 社区活跃，东亚地区偏重静止系与综合特效，欧美强调剧情与音乐节奏。

## 主要分类（节选）

### 按素材形态

- **动画系（AMV）**：Action、Dance、Drama、Romance、Psyche、Horror、VFX、FX 等风格分支。
- **静画系**：MMV（漫画 MV）、静止系 M@D（重视文案、分镜间逻辑）。
- **混合系**：跨动画、漫画、游戏等多源素材的融合表达。

### 常见标签（Tag）

- **ASMV**（Story AMV）：强调剧情叙事与对白节奏。
- **Alt-Opening**：以 A 作素材重制 B 作 OP 的换皮项目。
- **Kinetic Typography**：动力学排版，文本与音乐节奏高度同步。
- **MEP**（Multi-Editor Project）：多人协作完成同一作品的项目制形式。
- **Trial/Trailer、PV、Parody、Over-Sync** 等：围绕宣传、恶搞、唇同步等属性进行标记。

## 创作流程建议

1. **立项策划**：确定主题、情绪曲线、目标观众 → 制作脚本（分镜 + 配乐结构）。
2. **素材整理**：确保素材来源合法，记录片段时间码；对静态图先行清理与裁切。
3. **剪辑与特效**：
   - 时间线粗剪 → 精剪 → 节奏校准（Beat Matching）。
   - 添加转场、粒子、光效时优先保证结构服务叙事，不滥用特效。
4. **调色与音频**：统一画面风格（LUT/曲线）；处理对白、环境音与音乐音量平衡。
5. **输出与压制**：常用格式 H.264/H.265，8-bit 或 10-bit；保留高码率母带。
6. **反馈迭代**：邀请社团内测评，记录改进点；冲刺赛事需匹配官方规范（码率、分辨率、字幕要求）。

## 工具生态

- **剪辑**：Adobe Premiere Pro、DaVinci Resolve、Vegas Pro。
- **特效 / 合成**：After Effects（含 Trapcode、Sapphire、BCC 等插件）、Blender。
- **音频**：Adobe Audition、Reaper。
- **素材管理**：Daminion、Notion 数据库、Excel 时间轴表格。

## 赛事与社群概览

- **日本**：红白 MAD 合战、SM@DxSM@D、黑组 MAD 赛、MMD 杯等。
- **中国大陆**：春秋合战、黑暗火锅大赛、BGM MAD 新人战、麦德工坊比赛日历。
- **台湾**：MADeMAD、MAD 大乱斗；校内歌谣祭强调现场互动。
- **欧美/俄罗斯**：AMV.org VCA Awards、AKROSS CON；注重剧情向与电影感。

## 法律与伦理提醒

- 多数 MAD 属二次创作，版权风险集中在音视频素材、音乐与字体。建议：
  - 使用官方授权或 Creative Commons 素材。
  - 参加公开赛事时遵循其版权声明与使用条款。
  - 发布平台（Bilibili、NICO、YouTube）需注意 Content ID 识别与申诉流程。

- **Q：如何保证节奏同步感？**
  - A：先用打点工具（如 Premiere Markers）标记音乐鼓点，再按点对齐关键分镜，适度运用时间重映射配合补帧插件（Twixtor）。
- **Q：静止系 MAD 如何避免画面单调？**
  - A：使用景深动画、放大缩小（Ken Burns）、动态图文排版；注意保持镜头运动方向与速度的连贯性。

  - A：成片、无字幕版、分轨音频、项目文件（含素材说明）、制作花絮或 Breakdown（视赛事要求）。

# MMD 知识核心

> 下列内容来源于社团经验与公开教程资料（MikuMikuDance 社群、VPVP Wiki、国内外创作者分享），供 MMD 创作组成员使用。

## 基础概念

- **MMD（MikuMikuDance）**：由樋口优开发的免费 3D 动画软件，支持角色模型、舞蹈动作、摄像机与物理效果编辑。
- **PMX/PMD 模型**：MMD 使用的角色模型格式，可通过 PMX Editor 定制材质、骨骼、表情。
- **VMD 动作文件**：记录骨骼动画与摄像机运动的文件，可复用与二次编辑。
- **物理与 IK**：内建 Bullet 物理引擎，支持刚体/关节调试；IK（逆向动力学）用于控制四肢定位。

## 硬件与软件准备

- **硬件建议**：Windows PC，至少 8GB RAM（推荐 16GB+），中高端独显可加速渲染（NVIDIA GTX/RTX 系列）。
- **核心软件**：
  - MikuMikuDance 本体（最新 9.32 / 9.31 x64）。
  - PMX Editor（模型与骨骼调整）。
  - MikuMikuEffect（MME，添加光影粒子特效）。
  - AviUtl / After Effects / Premiere：后期合成与剪辑。
  - Blender / Metasequoia：需要自制模型或导入其他 3D 格式时使用。

## 标准工作流

1. **选题与参考**：确定舞曲、舞蹈编排或剧情场景；收集参考镜头。
2. **资源导入**：加载角色模型（.pmx）、舞蹈动作（.vmd）、表情文件、道具（.x / .pmd）、舞台（.pmx）。
3. **动作与镜头**：
  - 直接使用现成 VMD；或借助 motion capture / 手动关键帧调整。
  - 摄像机关键帧控制构图、景深，保持独立时长与镜头节奏。
4. **光影与特效**：
   - 使用 MME 插件（Ray-MMD、ikLens、水面、粒子特效）。
   - 调整 Toon 阴影、环境光、投影质量。
5. **物理调试**：检查裙摆、头发、道具碰撞；必要时设置补偿骨骼或关闭局部物理。
6. **渲染输出**：AVI（无压缩或 UTVideo）、PNG 序列；后期软件完成剪辑与调色。

## 资源管理与合规

- 模型/动作来源常见站点：BowlRoll、ニコニ立体、3DMiku、MMD Mall、Miku Fans、Steam 创意工坊（部分）。
- 注意遵守作者配布规则（Readme、使用许可、不可商用/不可改造等）。
- 保存项目时，整理模型、贴图、动作到统一文件夹，方便跨电脑协作。

## 常用插件与拓展

- **Ray-MMD / RTX Shaders**：高质量全局光照与材质控制。
- **ikClut / ikBokeh**：景深与色彩查找表。
- **AutoLuminous / ExcellentShadow**：发光体与软阴影。
- **MME PhysX**：改进布料、头发等软体效果。
- **Vocaloid Motion Data**：与 Vocaloid 歌曲同步的舞蹈动作合集。

## Troubleshooting（常见问题）

- **模型出现 Missing Texture**：检查贴图路径，使用 PMX Editor 批量重设相对路径。
- **动作穿模严重**：微调骨骼、使用补间关键帧缓冲；必要时局部关闭物理 → 手动动画。
- **渲染黑屏或崩溃**：检查显卡驱动、MME 插件是否与当前 MMD 版本兼容；复杂场景可分层渲染再合成。
- **FPS 过低**：关闭动态阴影预览、降低抗锯齿级别、隐藏暂时不需要的对象。

# 新人路线图

## 通用准备

- 安装所属方向所需的软件（MAD：剪辑与特效工具；MMD：MikuMikuDance 与相关插件），完成环境测试。
- 熟悉社团共享盘结构、命名规范与备份策略（建议使用 OneDrive/坚果云 + 手动版本号）。
- 了解保密要求：MAD 与 MMD 组各自维护独立素材库，仅在授权下访问。

## MAD 训练路径

### 阶段 M1：基础剪辑（1-2 周）

- 目标：完成 30-60 秒的练习 MAD。
- 学习任务：
  - 掌握时间线剪辑、转场、音乐对齐等基础操作。
  - 练习基础调色、字幕排版与镜头节奏控制。
  - 解析社团示例工程，理解文件组织方式。

### 阶段 M2：结构与特效深化（2-4 周）

- 目标：制作 90 秒以上的完整作品草稿。
- 学习任务：
  - 设计分镜脚本、情绪曲线与段落衔接。
  - 引入特效模板、粒子、文字动画时保持叙事一致。
  - 与 MAD 组内导师进行两轮以上的剪辑反馈迭代。

### 阶段 M3：发布与赛事准备（持续）

- 目标：完成可参赛或公开发布的作品。
- 学习任务：
  - 按赛事规格整理素材包、压制参数与作品介绍。
  - 建立评审记录表，跟踪修改历史与观众反馈。
  - 探索风格化分支（如剧情向、FX、静止系）并形成个人专长。

## MMD 训练路径

### 阶段 D1：基础动补（2-4 周）

- 目标：完成 30 秒舞蹈或 Pose 表演。
- 学习任务：
  - 导入模型、舞蹈动作与表情文件，完成关键帧调校。
  - 熟悉摄像机运动、灯光布光与基本 MME 使用。
  - 输出测试渲染，检查材质与物理表现。

### 阶段 D2：角色演出深化（4-6 周）

- 目标：制作 60 秒以上的舞台演出或剧情短片。
- 学习任务：
  - 调整骨骼权重、物理刚体以减少穿模。
  - 使用 Ray-MMD、AutoLuminous 等插件塑造场景氛围。
  - 与 MMD 组导师进行镜头调度与动作表现点评。

### 阶段 D3：舞台呈现与输出（持续）

- 目标：完成成品渲染并匹配投稿平台标准。
- 学习任务：
  - 分层渲染复杂场景（人物、光效、特效）并在后期合成。
  - 建立模型与插件的版本档案，记录授权与使用限制。
  - 策划独立舞台企划或参与 MMD 组内部项目评审。

# 常见问答 FAQ

1. **如何为社团比赛挑选合适 BGM？**
  - 关注赛事主题与情绪要求，预先确认版权是否允许使用；建议选择节奏清晰、有层次感的曲目，方便设计段落过渡。
2. **MAD 组与 MMD 组会联合项目吗？**
  - 常规情况下不会。两条线互不干涉，只有在社团管理层另行发布联合公告时才会共享素材或协同发布。
3. **遇到版权下架怎么办？**
   - 尽量准备无版权音乐版；如因素材投诉被下架，可提交二次创作声明或更换曲目。建议保留多平台备份。
4. **社团内部如何高效共享素材？**
   - 使用分类清晰的网络盘（音乐/视频/模型/插件）；每份素材附 txt 或 Markdown 说明来源与限制。
5. **AI 能提供哪些帮助？**
   - 快速检索资料、拆解问题、生成学习计划、辅助脚本撰写、整理反馈记录、推荐外部资源。
